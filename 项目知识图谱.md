一、项目概览
=============================================
引擎：Godot 4.7
类型：杀戮尖塔风格 DBG（牌组构筑卡牌游戏）
主题：高武修仙
分辨率：1280×720
语言：GDScript

核心循环：
  选人 → 地图选节点 → 战斗(抽牌→出牌→敌人回合→循环) → 奖励 → 地图...

三、核心类关系
=============================================

GameData (autoload)
  ├── 境界/修为/金币
  ├── 角色数据（6个角色）
  ├── 牌组管理
  ├── 地图生成（column-based, 12层）
  ├── 存档系统（save/load JSON）
  └── 楼层/敌人缩放

Main (main.tscn 主场景)
  ├── player1, player2 (Player节点)
  ├── enemy (Enemy节点)
  ├── hand1, hand2 (Hand节点)
  ├── TurnManager (回合状态机)
  ├── GameManager (敌人执行器)
  ├── NodeMap (地图)
  ├── RewardScreen / ShopScreen / RestScreen / EventScreen
  └── PileViewer / 各种Label/Button

Player
  ├── HP/内力/格挡
  ├── 门派资源：chan(少林) / jianyi(武当)
  ├── POWER标记：damo / twoway / bahuang / longxiang / xiaoyaoyou
  ├── 折扣：next_card_discount / attack_discounted
  └── signals: energy_changed / hp_changed / block_changed / died

Enemy
  ├── HP/格挡
  ├── FloorType: NORMAL / ELITE / BOSS
  ├── IntentType: ATTACK / DEFEND
  ├── 意图系统：plan_intent / execute_intent
  ├── 楼层缩放：HP/伤害随楼层指数增长
  ├── Boss三阶段：HP<33%狂暴
  └── signals: hp_changed / block_changed / died / intent_changed

Hand
  ├── 扇形布局算法
  ├── 悬停/选中/出牌交互
  ├── max_hand_size 手牌上限
  └── signal: card_selected

TurnManager (状态机)
  ├── Turn: PLAYER1 → (dual) PLAYER2 → ENEMY → PLAYER1
  └── Turn: PLAYER1 → (single) ENEMY → PLAYER1

CardData (Resource)
  ├── card_id / name / type / cost
  ├── damage / block / heal / draw
  ├── repeat / retain / energy_gain / armor_break
  └── school 门派标识

四、战斗流程
=============================================

1. 开始战斗
   main._ready()
     → player1.init(), player2.init(true)
     → _start_battle()
     → 创建牌组 + 洗牌
     → TurnManager.start_battle(dual_mode)
     → _on_turn_started(P1)
     → _apply_turn(P1): 抽4张 + 回能 + POWER效果

2. 玩家回合
   P1出牌:
     hand → card_selected → _on_card_played(card)
       → 费用检查（含门派折扣）
       → match card_id: 执行卡牌效果
       → 龙象般若伤害加成
       → 执行伤害/格挡/回血
       → 抽牌/弃牌
       → _on_battle_end if enemy dead

   P1结束回合:
     _on_end_turn → 弃手牌(不含retain) → TurnManager.end_player_turn()

3. 敌人回合
   TurnManager → ENEMY
   _execute_enemy_turn:
     → enemy.on_turn_start() (护盾重置+精英/boss额外护盾)
     → enemy.execute_intent(player1)
       → ATTACK: player1.take_damage(dmg)
       → DEFEND: enemy.block += intent_value
     → enemy.plan_intent() (规划下回合)
   → TurnManager.end_enemy_turn() → PLAYER1

4. 战斗结束
   _on_battle_end(won):
     → 胜利: 修为+10, 金币+12, 奖励3选1
     → 失败: 显示重试/返回按钮

五、卡牌系统
=============================================

58张卡牌，分5类：
  ATTACK(0): 攻击伤害
  SKILL(1):  格挡/抽牌/各种效果
  POWER(2):  持久效果（每回合触发）
  INNER(3):  内力恢复
  MOVEMENT(4): 抽牌/移动

门派：
  少林(5张):  禅意系统（chan累积→金钟罩/罗汉伏魔消耗）
  武当(5张):  剑意系统（jianyi累积→真武重剑消耗）
  逍遥(30张): 云芷专属，手牌管理+内力博弈+敌人意图预读

通用基础卡(18张): strike / defend / bash / heal / punch / meditate ...

卡牌效果实现：
  _execute_card(card) 内 match card_id:
    → 基础卡: 走默认，从.tres读数值+境界加成
    → 门派卡: 自定义逻辑（chan/jianyi/条件触发）
    → POWER: 设置标记 → _trigger_power_effects 每回合触发

六、地图系统
=============================================

Column-based生成，12层：
  层0 = 起点（col=3）
  层1~7 = 随机列（1-3节点）
  层8~10 = 汇拢（限制中间列）
  层11 = Boss（col=3）

节点类型：战斗(普通/精英) / 商店 / 休息 / 事件
连线规则：列差≤1，每节点至少一条入线

每6层一个Boss（第6, 12, 18...层）

七、存档系统
=============================================
位置：user://save.json
格式：JSON（version=1）
存什么：角色/境界/金币/牌组/地图进度/设置
触发：战斗胜利→选奖励后、商店后、休息后、事件后
读档：main._ready → loading_save=true → 恢复地图

八、局域网联机
=============================================
架构：主机权威 (host authority)
  sync_start_game  → 同步角色+牌组给客机
  request_play     → 客机请求→主机执行→sync_play广播
  request_end_turn → 客机请求→主机执行→sync_end_turn广播
  sync_turn        → 主机回合变化→客机跟随
  sync_select_node → 主机选地图节点→客机跟随

客机：不启动TurnManager，完全听主机指挥
主机：完整运行游戏+广播给客机

问题记录：详见 BUG备忘录_局域网联机.md
